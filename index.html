Background Subtraction · 01

Median Filter

Imagine you're filming a busy road. You want to know what the road looks like without any cars. The Median Filter collects a stack of n frames and, for each pixel, takes the middle (median) value. Since cars move, the road itself appears most of the time — so it "wins" the median, and cars get filtered out.

📸
Estimated Background
Median of n frames per pixel. n=20 gives a rough background; n=50 gives a cleaner, more stable one.
🎭
Foreground Mask
Pixels that differ from the background model beyond a threshold Th are marked as foreground (moving objects).
🔢
Detection Rule
|I(x,y,t) − B(x,y,t)| > Th
If true → foreground; else → background.

How n affects quality

✦ Higher n (e.g. 50)
  • Cleaner background estimate
  • More frames to average over
  • Better handles slow objects
✦ Lower n (e.g. 20)
  • Noisier background
  • Faster to compute
  • Less memory needed

Background Subtraction · 02

Advantages vs. Limits

These simple methods (median, mean, frame differencing) are like a simple security guard who only knows "this is what the room normally looks like." They work great in ideal conditions, but fail if the lighting changes, objects move slowly, or there's a bimodal background (like a flickering screen or waving trees).

ADVANTAGES
  • Extremely easy to implement
  • Fast computation
  • Background model adapts over time
DISADVANTAGES
  • Frame differencing: accuracy depends on object speed + frame rate
  • Mean/median: high memory needs
  • Single global threshold Th — not adaptive to time
  • Fails with bimodal background
  • Fails with slow-moving objects
  • Fails if lighting changes over time

The Core Problem

|I(x,y,t) − B(x,y,t)| > Th
⚡
Problem 1: One Global Th
The same threshold for all pixels. A dark corner and a bright window get the same Th — that's a bad idea.
⏰
Problem 2: Th is Fixed
Th doesn't change with time t. If the sun sets and the whole scene darkens, Th can't adapt.

Background Subtraction · 03

Gaussian Mixture Model (GMM)

Imagine each pixel as a person with a memory. Instead of just "what's the average color?", they remember several typical color states — like "usually it's the gray road, sometimes a car's shadow passes." Each state is a Gaussian bell curve. The model picks whichever Gaussian best matches the current pixel and uses the most stable, frequent ones as the background.

🔔
Why a Mixture?
Multiple surfaces can appear at a single pixel over time (road, car shadow, pedestrian). Each gets its own Gaussian.
🌅
Why Adaptive?
Lighting conditions change. The Gaussians update their mean, variance, and weight as new frames arrive.
🏷️
Background Decision
Gaussians with high weight + low variance = background. They appear often and stay stable.

Update Rules

1
New pixel arrives. Check if it matches an existing Gaussian (within 2.5σ).
2
If match found: update that Gaussian's mean, variance, and weight.
3
If no match: replace the weakest Gaussian with a new one at the pixel value.
4
Rank Gaussians by weight/variance ratio. Top ones = background.
5
Pixel doesn't match background Gaussians → classified as foreground.

Gaussian Reminder

𝒩(x|μ, σ²) = 1/√(2πσ²) · exp(−(x−μ)²/2σ²)

Background Subtraction · 04

KNN Background Modeling

KNN is like asking your neighbors "is this normal?" For each pixel, you store the last n observed values. When a new pixel arrives, you find its K nearest historical values. If enough neighbors are close in color → it looks like background. If the new value is a stranger → it's foreground.

📚
Non-Parametric
No assumption about distribution shape. More flexible than GMM which assumes Gaussian shape.
📏
Distance-Based
Uses Euclidean distance to find K nearest neighbors from the stored history of n pixel values.
🔄
Adaptive Updates
Old samples are replaced by new observations, so the model adapts to gradual scene changes.

GAN — Generative Adversarial Network

A GAN is like a forger vs. a detective. The forger (Generator G) keeps making fake paintings. The detective (Discriminator D) tries to spot the fakes. They train each other — the forger gets better at forging, the detective gets better at spotting — until the forgeries are indistinguishable from real art.

Flip Cards — Test Your Memory

🎨
Generator G
Input: random noise z~N(0,I)
Output: fake image G(z)
Goal: fool D so D(G(z))=1
Architecture: FC→Reshape→ConvTranspose
🔍
Discriminator D
Input: real x or fake G(z)
Output: probability D(x)∈[0,1]
Goal: correctly classify real vs fake
Architecture: Conv→BN→LeakyReLU
📊
Loss Function
min_G max_D V(D,G)
D maximizes: log D(x) + log(1−D(G(z)))
G minimizes: log(1−D(G(z)))
V(D,G) = 𝔼[log D(x)] + 𝔼[log(1 − D(G(z)))]

Generative Models · 09

GAN Variants

🏗️
DCGAN
Uses Conv/Transposed-Conv instead of FC layers. BatchNorm + LeakyReLU for stable training. Widely used baseline for image generation.
🏷️
cGAN
Adds a label/condition to both G and D. Enables controlled generation — you can say "generate a dog" and it does exactly that.
🔄
CycleGAN
Image-to-image translation without paired data. Uses cycle consistency: A→B→A must reconstruct original. Example: horse ↔ zebra.
✏️
pix2pix
Supervised translation with paired data. Uses GAN loss + L1 loss. Example: sketch → photo, edges → realistic image.
💎
StyleGAN
Uses a mapping network and Adaptive Instance Normalization. Controls coarse (pose) to fine (texture) detail. Produces hyper-realistic faces.

VAE — Variational Autoencoder

A regular autoencoder compresses images to a fixed point. A VAE instead compresses to a fuzzy region (a probability distribution). By sampling from that region, you can generate new, similar images. It's like describing a face not by exact coordinates but by "roughly this shape, give or take."

Forces the latent distribution to be close to 𝒩(0,1). This makes the latent space smooth and enables interpolation.

Diffusion Models

Imagine shredding a photo by adding sand grain by grain until you just have noise. A diffusion model learns to reverse this — starting from pure noise, it repeatedly removes a little sand until a clear image appears. Stable Diffusion does this in a compressed "latent space" for efficiency.

→
Forward Diffusion: Gradually add Gaussian noise to an image over T timesteps until it's pure noise.
←
Reverse Diffusion: A neural network (U-Net) learns to predict and remove noise step-by-step.
🎲
Generation: Start from random noise → iteratively denoise → realistic image.

Motion & Flow · 12

Optical Flow

Optical flow is like tracking where every raindrop on your windshield moves between two consecutive photos. For every pixel in frame t, you estimate a motion vector (u,v) showing where it went in frame t+1.

📡
What It Is
A 2D vector field (u,v) representing apparent pixel motion between two frames. Caused by camera motion, object motion, or both.
🌑
Key Assumption
Brightness Constancy: a pixel's intensity doesn't change as it moves. I(x,y,t) = I(x+u, y+v, t+1).
❓
Aperture Problem
One equation per pixel, two unknowns (u,v). You can't tell direction through a small window — need extra constraints.

Optical Flow Equation

Iₓ·u + Iᵧ·v + Iₜ = 0    (1 equation, 2 unknowns)

Applications

🎬
Video Compression
Encode only the motion, not full frames.
🤸
Action Recognition
Motion patterns reveal human activities.
🚗
Autonomous Driving
Estimate egomotion and obstacle motion.

Lucas-Kanade Optical Flow

LK says: "My pixel and all its neighbors moved together (spatial coherence)." By assuming a small patch moves as a rigid block, you get 25 equations (in a 5×5 window) for 2 unknowns — now you can solve it with least squares!

The Math

Assume spatial coherence in a 5×5 window → 25 equations:

A·d = b     (25×2 matrix · 2×1 vector = 25×1 vector)

Solution: d = (AᵀA)⁻¹ Aᵀ b

When Is It Solvable?

✅
Corners work!
AᵀA must be invertible. Both eigenvalues λ₁ and λ₂ must be large → this is exactly a corner! Connected to Harris corner detector.
❌
Edges fail
λ₁ >> λ₂ → edge. One direction is ambiguous. Aperture problem persists.
❌
Flat regions fail
Both eigenvalues near zero → no texture, no gradient information, cannot estimate flow.

FlowNet — Deep Optical Flow

Traditional optical flow requires hand-crafted assumptions (brightness constancy, spatial coherence). FlowNet throws away the rulebook and just feeds a CNN two consecutive frames, letting it learn what motion looks like from millions of examples.

🔗
FlowNetSimple (FlowNetS)
Stacks both frames as a 6-channel tensor (3+3 RGB). Processes them jointly through a standard encoder-decoder CNN.
🧩
FlowNetCorr (FlowNetC)
Processes each image separately, then uses a correlation layer to explicitly match feature vectors at different locations.

Impact & Legacy

1
First deep learning model to treat optical flow as supervised learning (2015).
2
Proved CNNs can learn both feature representation and correspondence matching.
3
Inspired PWC-Net, RAFT, and the entire modern deep flow estimation field.
4
Applications: video analysis, robotics, autonomous driving.
